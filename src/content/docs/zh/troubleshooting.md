# 故障排查

## 设备相互看不到

按顺序排查，绝大多数情况会落在前三条：

1. **同一网络？** 两端必须在同一 LAN 段。一台用 Wi-Fi、一台用蜂窝网络是不可能互通的。一台连 5 GHz SSID、一台连 2.4 GHz SSID 通常可以，但前提是路由器把两者桥接起来。
2. **AP 隔离？** 酒店、咖啡馆、会议 Wi-Fi 经常开启 AP 隔离，会阻断所有点对点流量。可改用个人热点，或通过 6 位短码侧带配对。
3. **防火墙？** 在系统防火墙里放行邻传：
   - **macOS** —— 系统设置 → 网络 → 防火墙 → 选项 → 允许 `Link2Send`。
   - **Windows** —— Windows Defender 防火墙在首次启动时会弹窗。如同时使用专用与公共网络，请都点"允许访问"。
4. **手动配对。** 如果发现被拦截但你能直接到达对方 IP，仍然可以扫码或输入 6 位短码完成配对。配对后邻传会沿用现有会话。

## "接收中…"卡住不动

如果发送端显示"已中断"而接收端仍卡在"接收中…"，多半是版本低于 v0.5.1。请把两端都升级。

v0.5.1 起，发送端连接掉线时接收端会在毫秒级别翻为"中断"。partial 文件与 manifest 会保留在磁盘上，下次重试可安全续传。

## 传输速度异常慢

- **Wi-Fi 质量。** 2.4 GHz 信道在邻居 SSID 拥挤时大文件传输可能掉到个位数 MB/s。请改用 5 GHz，或至少一端使用有线。
- **杀毒软件实时扫描。** 部分企业 EDR 会对每个落盘 chunk 做检查，吞吐可能减半。可信任 peer 时，可把接收目录加入白名单。
- **磁盘速度。** 接收到慢的外置硬盘上时整体速度会被磁盘写入速度卡住。如有条件请先收到本地 SSD。

## macOS 首次打开告警

0.5.1 暂未完成 Apple 公证。首次打开请：

1. 在访达中右键点击 App → **打开**。
2. 在确认弹窗中再次点击 **打开**。

macOS 会记住该选择，之后双击即可正常启动。

## Windows 首次安装提示

Windows SmartScreen 可能在首次安装时弹出提示。点击 **更多信息** → **仍要运行** 即可。安装包已使用我们的发行证书签名，微软的信誉系统在累计若干次安装后会自动取消该提示。

## 文件完整性

怀疑文件损坏时：

- 在 [下载页](/zh/download/) 比对 SHA-256 校验值（可用时）。
- 用 SHA-256 比对原文件与接收文件。v0.5.1 每个 chunk 都会校验，接收损坏极少出现，通常是某一端磁盘问题。

## 还是没解决？

请发邮件至 **support@link2send.com**，并附：

- 两端的邻传版本（设置 → 关于）。
- 各自的操作系统及版本。
- 简要描述你已尝试的步骤。

应用内 **网络诊断** 工具会生成一份小报告，可一并附上。
