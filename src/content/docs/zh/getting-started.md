# 快速开始

本指南帮你在两台设备上安装邻传、完成配对并发出第一份文件。

## 1. 在两台设备上安装

1. 打开各自设备上的 [下载页](/zh/download/)。
2. 选择对应平台的安装包：
   - **macOS** → `邻传_0.5.1_x64.dmg`（Intel 构建；Apple Silicon 通过 Rosetta 运行）
   - **Windows** → `邻传_0.5.1_x64-setup.exe`
   - **iOS / Android** → 应用商店审核中；可在下载页留邮箱以获取上线通知
3. 安装并启动邻传。首次启动会在本地生成唯一的设备证书——无需注册，无需邮箱。

## 2. 把两台设备接入同一网络

邻传只在 LAN 上通信。请确保两台设备处于 **同一 Wi-Fi 或同一以太网段**。酒店和会议 Wi-Fi 常常禁用点对点流量，必要时可改用个人热点或有线网络。

## 3. 配对

同一 LAN 上的设备会在几秒内通过 UDP 自动发现，对方设备会出现在联系人面板。

首次连接需要确认配对，任选一种方式：

- **二维码** —— 用一台设备扫描另一台显示的二维码。
- **短码** —— 在一台上输入另一台显示的 6 位短码。

配对会锁定对端的证书指纹，之后的连接会自动校验，无需再次确认。

## 4. 开始发送

配对后即可：

- **拖拽** 文件或文件夹到聊天窗口
- 点击 **回形针** 浏览并选择文件
- **粘贴** 剪贴板里的图片或截图
- 输入并发送 **文字、表情、链接**

接收文件默认保存到 `~/Downloads/Link2Send/`，可在设置中修改。

## 底层发生了什么

- 每条会话基于 **会话级 TLS 连接池**（4–8 条长连接 slot）。
- ≤ 50 MB 的文件占一条 slot 跑完；大文件分片到最多 4 条 slot 并行。
- 每个分块都做校验，连接中断时下一次会从最后一个已确认的块继续。
- 聊天历史所在的本地数据库使用 AES-256-GCM 加密。

如果遇到网络问题，请前往 [故障排查](/zh/docs/troubleshooting/)。
